# CSS技术发展

简单总结下现代CSS框架，大纲如下

- CSS框架的起源
- CSS框架的发展
- CSS框架的现状

## 起源

CSS(Cascading Style Sheets)用来描述HTML，XML文本的呈现，用来设置网页的样式和布局。类似声明式编程，我们在css中声明样式，然后浏览器的渲染引擎就会根据css来渲染对应的效果，具体如何渲染不用我们关心。

这是最基本的方式，也是浏览器能够理解的东西。后续的所有框架的操作，最终都是要落到这一步上面来实现。

但是当涉及到大型软件工程的时候，由于CSS原生语法的缺陷，无法实现变量、嵌套、继承等等编程特性，且各大浏览器厂商对于CSS的支持标准不统一，让开发者编写CSS成了一件非常痛苦的事情。

## CSS预处理器的诞生

为了解决这些问题，CSS 预处理器的概念诞生了，让开发者先写预处理器的代码，然后通过预处理器的编译器，转译成能在浏览器上运行的CSS代码，诸如此类的产品有sass，less。通过扩展CSS语法，来实现更高级的功能。

同时在后面也出现了PostCSS这类转译工具，将预处理器的语法转换成CSS的同时，还会对现有CSS代码进行优化，比如自动添加浏览器前缀和压缩CSS等

## 响应式布局和flexbox

flexbox是2012年才被W3C推荐使用，在此之前，响应式布局早就是各个网站的刚需。这也催化了一批CSS框架的诞生，如推特的Bootstrap(2011年)这类框架，他们提供了栅格系统来创建响应式布局，方便在移动设备上也能很好地展示web，同时还标准化了样式，提供了一系列预设好的样式，像按钮、表单、导航、toast等等。

## postCSS和插件系统

postCSS(2013)，带来船新的一套插件系统，将工作流引入到了CSS当中，让用户可以随意转换CSS，无论CSS怎么发展，postCSS都只需要换个插件就行了，PostCSS 本身只是一个将 CSS 解析为抽象语法树（AST）和再将 AST 重新转换为字符串的工具。所有的 CSS 转换和优化工作都是由插件完成的。基于postCSS这种高度自定制的工具，自动加上浏览器前缀，优化压缩CSS代码等等骚操作

## CSS Modules

css modules(2014)将CSS转换成了JS模块，在JS文件中可以引入，并且应用某个样式，从而非常方便地实现了样式的作用域隔离，以及全局作用域，变量，继承等特性。同时CSSModules还支持各种预处理器，能够很好地兼容。

## Tailwind CSS 现代CSS框架

Tailwind(2017)提供了低层次的、可组合的 CSS 类，使得开发者可以在 HTML 中直接构建 UI，而不需要写额外的 CSS。Tailwind集成了前面很多CSS框架，CSS库的功能，支持响应式布局，支持插件系统，高度可自制等等

## 设计系统

再到后面，CSS的轮子几乎已经造得差不多了，开发人员痛点也多来自于其他地方。随着用户对美观的要求逐渐提高，也诞生了UX/UI这类职业，前端们常常要根据设计稿来实现对应效果，一个致命的问题就是设计的准则不统一， 导致每次都是要手搓CSS代码。级而也诞生一个问题就是写的组件库也经常不能够满足设计要求，需要重写，因此众多大公司都会出一套`Design Laguage System`来统一设计标准，且减轻前端工程师的负担，
